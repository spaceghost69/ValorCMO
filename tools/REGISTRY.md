# Tool Registry for Imported Marketing Skills

This local registry exists to make the imported marketing skills OpenClaw-friendly in this workspace.

## How to use these references
- Treat each integration file as a lightweight operating note, not proof that the integration is already configured.
- If a platform is available in this environment, use it.
- If it is not configured or auth is missing, produce a useful draft, checklist, mapping, or implementation plan instead of blocking.
- Prefer `.agents/product-marketing-context.md` for shared product context.

## Current stance
- OpenClaw is the orchestration layer.
- Local workspace files are draft, cache, bootstrap, or fallback.
- Durable business records should land in api-0 / GrayMatter when appropriate.
- Do not fake live integration state. Say clearly when a tool is not connected.
